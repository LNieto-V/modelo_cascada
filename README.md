# Proyecto: Editor de Procesos Standalone según ISO/IEC/IEEE 24774
Implementación de software utilizando la metodología de desarrollo en Cascada (Waterfall) basada en los procesos del estándar ISO/IEC/IEEE 12207.

## Estructura del Proyecto
El ciclo de vida en cascada se divide en las siguientes etapas documentadas y estructuradas:

1. **[Análisis de Requerimientos](file:///c:/Users/lnieto/modelo_cascada/An%C3%A1lisis%20de%20requerimientos/)**:
   * **Proceso (`proceso.md`):** Definición del proceso de elicitación y definición de requisitos conforme a las cláusulas 6.4.2 y 6.4.3 de ISO/IEC/IEEE 12207.
   * **Línea Base 1 (`outcomes/resultado1.md`):** Entregables del Proceso de Definición de Requisitos y Necesidades de los Interesados (StRS 6.4.2) - Aprobado y Congelado.
   * **Línea Base 2 (`outcomes/resultado2.md`):** Especificación de Requisitos del Sistema (SyRS 6.4.3) - Aprobada y Congelada.
2. **[Arquitectura](file:///c:/Users/lnieto/modelo_cascada/Arquitectura/)**:
   * **Proceso (`proceso.md`):** Descripción del proceso de definición de arquitectura (Cláusula 6.4.4).
   * **Línea Base (`outcomes/resultado1.md`):** Especificación de Arquitectura del Sistema (SAD).
3. **[Diseño](file:///c:/Users/lnieto/modelo_cascada/Diseño/)**:
   * **Proceso (`proceso.md`):** Descripción del proceso de diseño detallado (Cláusula 6.4.5).
   * **Línea Base (`outcomes/resultado1.md`):** Especificación de Diseño Detallado.
4. **[Construcción](file:///c:/Users/lnieto/modelo_cascada/Construcci%C3%B3n/)**:
   * Codificación y pruebas unitarias de los módulos del editor.
5. **[Verificación y Validación](file:///c:/Users/lnieto/modelo_cascada/Verificaci%C3%B3n%20y%20validaci%C3%B3n/)**:
   * Plan de pruebas, casos de prueba y reportes de conformidad.
